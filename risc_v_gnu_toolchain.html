RISC-V GNU Toolchain (RV32I) Setup in WSL

[1] Prepare Environment

Explanation: Install required packages for building the RISC-V GNU Toolchain.

sudo apt update
sudo apt install -y build-essential git libgmp-dev libmpfr-dev libmpc-dev texinfo bison flex libtool patchutils bc zlib1g-dev libexpat-dev

[2] Clone the Repository

Clone the repository in WSL home directory (case-sensitive filesystem) to avoid build errors.

cd ~
git clone https://github.com/riscv/riscv-gnu-toolchain riscv-gnu-toolchain-rv32i
cd riscv-gnu-toolchain-rv32i

[3] Keep Build Files Separate

Create a separate directory for build files to keep them isolated from source files.

mkdir build && cd build

[4] Configure the Toolchain

Configure the toolchain with --prefix for the install path, --with-arch to set RV32I, and --with-abi to set ILP32.

../configure --prefix=/opt/riscv32i --with-arch=rv32i --with-abi=ilp32

[5] Build the Toolchain

Build the toolchain, using multiple CPU cores for faster compilation.

make -j$(nproc)

[6] Install the Toolchain

Install the compiled toolchain to the specified directory.

sudo make install

[7] Verify Installation

Check that the toolchain is configured with --with-arch=rv32i and --with-abi=ilp32.

riscv32-unknown-elf-gcc -v
ls /opt/riscv32i/bin

Expected output includes tools like:

riscv32-unknown-elf-gcc
riscv32-unknown-elf-as
riscv32-unknown-elf-ld
riscv32-unknown-elf-objdump

[8] Add Toolchain to PATH

Add the toolchain to your PATH permanently.

echo 'export PATH=/opt/riscv32i/bin:$PATH' >> ~/.bashrc
source ~/.bashrc
